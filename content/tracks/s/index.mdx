## What & why

If you ship on Java/Spring Boot you no longer need to leave your stack for AI work. Spring
AI 2.0 hit GA June 12, 2026 — full MCP, tool calling, memory, vector stores, all native.
Concepts transfer both ways with the Python tracks.

## What you must cover

- `ChatClient` — model-agnostic LLM calls (swap Ollama ⇄ frontier via config)
- Tool calling via `@Tool` annotations
- Structured output (POJO binding, validation)
- `ChatMemory` — stateful conversation
- `VectorStore` abstraction (pgvector backend — fits Postgres)
- Advisors API (RAG, logging, guardrails as pluggable advisors)
- MCP server via `@McpTool` annotations (`spring-ai-starter-mcp-server-webmvc`)
- MCP client + MCP Security (OAuth/RBAC)
- (compare) LangChain4j as the framework-agnostic alternative

## Version warning (verbatim from spec)

Spring AI ships fast; 1.0→1.1 broke APIs. Pin versions via the Spring AI BOM, read
changelogs every upgrade. MCP spec breaking change ~July 28, 2026 (session removal) —
build MCP servers against the post-change spec.

## Java framework menu (verbatim from spec)
